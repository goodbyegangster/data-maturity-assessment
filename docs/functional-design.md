# functional-design

## 画面設計

### トップページ (データ成熟度評価モデルの選択)

- Input Parameters
  - なし
- 表示情報
  - 本サービスの説明
  - LICENSE 情報
    - [../README.md](../README.md) の `ライセンスと帰属 (License & Attribution)` を参照
  - GitHub Repository [data-maturity-assessment](https://github.com/goodbyegangster/data-maturity-assessment/tree/main) のリンク
- 操作
  - 利用する評価モデルの選択
    - 表示順は各評価モデルの `.modelId` の昇順
    - 初期状態は未選択
    - 現在選択されている評価モデルの `.modelDescription` が表示される（初期状態は空欄となる）
  - 測定の開始
    - `測定画面` へ遷移
    - 評価モデルの未選択の場合は、エラーメッセージを表示

### 測定画面

- Input Parameters
  - 利用するデータ成熟度評価モデル
- 表示情報
  - 回答状況バー
    - 全質問数に対する回答済み数を表示
  - `.assessments[]` に記載の質問および回答選択肢
    - `.assessments[].topic` 毎のアコーディオン表示（初期状態は非 open とする）
    - 選択した `levels` に応じて、radio 部分の背景色を `配色ルール` を踏まえ変更する
- 操作
  - `.topicAreas[]` の選択
    - 選択した `.topicAreas[]` にあたる `.assessments[]` が表示される
    - 初期状態は全選択
    - 複数選択可
    - 未選択不可 (選択解除操作をした場合、エラーメッセージを表示)
    - 回答途中で `.topicAreas[]` を追加/削除した場合でも、既存の回答内容を維持させる
  - 質問への回答
    - 単 1 回答
    - 初期状態は未選択
  - 結果の表示
    - `結果表示画面` へ遷移
    - 未回答がある場合、そのまま測定を実施してよいか訊ねるメッセージを表示する
      - 未回答のまま測定に進む場合、`.assessments[].levels` の最小値として扱う
  - 評価モデル選択画面へ戻る
    - `トップページ (データ成熟度評価モデルの選択)` へ遷移
    - 回答内容はリセットされる

### 結果表示画面

- Input Parameters
  - 利用したデータ成熟度評価モデル
  - 選択した `.topicAreas[]`
  - 質問回答結果
- 表示情報
  - 回答結果のレーダーチャート
    - 詳細は後述の `レーダーチャート設計` を参照
  - 回答結果一覧
    - `.assessments[].topic` 毎のアコーディオン表示（初期状態は非 open とする）
    - 回答結果の `levels` に応じて、表示画面の背景色を `配色ルール` を踏まえ変更する
- 操作
  - 結果画面の PDF 取得機能
    - ブラウザの印刷機能 (`window.print()`) を利用した PDF 保存
    - 印刷時は回答結果一覧のアコーディオンをすべて展開し、印刷後に元の状態に戻す
    - PDF 保存ボタンは印刷対象から除外する
  - 評価モデル選択画面へ戻る
    - `トップページ (データ成熟度評価モデルの選択)` へ遷移

`結果表示画面` からは `測定画面` へは遷移させない。つまり、回答を修正して再測定といった動作には対応しない。

## レーダーチャート設計

### スコアの求め方

- `.assessments[].topic` ごとの回答された `.assessments[].levels の key (整数値)` の平均値
- 小数点以下は切り捨て

### レーダーチャートの定義

- 軸
  - `.topicAreas[]`
  - 選択されなかった `.topicAreas[]` の項目は、未選択であることを分かりやすく表現するため、軸ラベルをグレーアウトする
  - 選択されなかった `.topicAreas[]` のスコア値は `.maturityLevels[]` の最小値として扱う
  - データ点（頂点）の色は、対応するトピックのスコアレベルを `配色ルール` に従って表示する
    - 未選択トピックのデータ点は灰色で表示する
- 目盛り
  - `.maturityLevels[]`
- 中心点
  - `.maturityLevels[]` の最小値より 1 を減算したもの
  - なお、中心点のラベルは表示しない

## 配色ルール

level に応じて、以下の配色を利用する。

- 0: 赤
- 1: オレンジ
- 2: 黄色
- 3: ライムグリーン
- 4: 緑
- 5: 青

level に該当する配色が定義されていない場合、灰色を利用する。

配色ルールは測定画面のラジオボタン、結果表示画面の回答結果一覧、およびレーダーチャートのデータ点色に適用する。

なお、レーダーチャートのポリゴン（塗りつぶし・枠線）は固定色（青）を使用する。

## エンドポイント設計

### ページエンドポイント

- `GET /`
  - トップページ
- `GET /assessment`
  - 測定画面
  - クエリパラメータ:
    - `model` (必須): `.model`
  - HX-Request ヘッダーあり: 測定画面の Partial のみ返す
  - HX-Request ヘッダーなし: フルページを返す
  - バリデーション: `model` が不正値の場合は `/` にリダイレクト
- `POST /result`
  - 結果表示画面
  - フォームデータ:
    - `model` (必須): `.model`
    - `topicAreas` (必須): 選択した `.topicAreas[].topic` のリスト
    - `answers[{ .assessments[].id: 回答値 }]` (必須): `.assessments[].id` をキーとした回答の `.assessments[].levels`
  - リロード・URL 直打ちは非対応（POST のみ到達可能）
    - リロード時はブラウザの「フォームを再送信しますか？」の挙動に委ねる

### Partial エンドポイント (htmx専用)

- `GET /partials/questions`
  - 測定画面のカテゴリ選択変更時に呼ばれる質問一覧の Partial
  - クエリパラメータ:
    - `model` (必須): `.model`
    - `topicAreas` (必須): 選択中の `.topicAreas[]` の識別子リスト
  - HX-Request ヘッダーなしの場合: `/assessment?model={model}` にリダイレクト
  - 返却: `.assessments[]` 一覧の HTML フラグメント
- `GET /partials/model-description`
  - トップページの評価モデル選択時に呼ばれる説明文の Partial
  - クエリパラメータ:
    - `model` (必須): `.model`
  - HX-Request ヘッダーなしの場合: `/` にリダイレクト
  - 返却: 評価モデル説明文の HTML フラグメント

### エラーレスポンス方針

- バリデーションエラー: `HX-Trigger` ヘッダーでフロントエンドに通知
- 不正なエンドポイントアクセス: 上記の通り各エンドポイントでリダイレクト
